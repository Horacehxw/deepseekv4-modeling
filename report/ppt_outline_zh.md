### 幻灯片 2：摘要 — 六项核心发现

- V4 将 KV 缓存缩减 4.6 倍（vs V3）：15,168 字节/token vs 70,272，1M token 时 KV 仅 6.9 GB
- 910C 赢得 Prefill，H20 赢得 Decode（分别快 1.2 倍和 3.4--6.1 倍），反映截然相反的硬件设计取舍
- mHC 内核融合是 910C 上杠杆率最高的单项优化：融合 SP+BF16 mHC 带来 7.6 倍端到端 Prefill 加速
- Decode 为带宽受限、MoE 主导：910C 上 MoE 占 70%，H20 AllToAll 通信占 27%
- 前缀缓存改变 P/D 比例：0% 缓存时约 1P:6D，99% 缓存时高达 1P:42--64D
- 128K--256K 上下文将瓶颈转移至 Lightning Index 注意力（256K 时超 46%）
- **可视化建议**：编号高亮卡片或图标网格，总结六项核心发现

## 第二部分：模型结构

### 幻灯片 3：DeepSeek V4 架构概览

- 43 层 Transformer：2 层全注意力（ratio=1）、21 层 C4A（4 倍压缩）、20 层 C128A（128 倍压缩）
- MQA：64 个 Q 头、1 个 KV 头、head_dim=512，Q LoRA rank=1,024
- MoE：256 个路由专家（top-6 路由）、1 个共享专家、inter_dim=2,048
- manifold Hyper Connection (mHC)：每个子层均有 FP32 前/后投影 + Sinkhorn 归一化，hc_mult=4
- Lightning Index：64 个索引头、dim=128、topK=512，用于压缩 KV 条目选择
- KV 压缩：分组投影对 K 和 V 缓存按比例压缩（C4A=4 倍、C128A=128 倍）
- **可视化建议**：架构框图，展示层类型、MoE 路由、mHC 数据流和 KV 压缩机制

### 幻灯片 20：行业启示 — 六大趋势

1. **KV 缓存管理进入新时代**：V4 使每 token KV 次线性增长；Mooncake、SGLang、vLLM v1 的分层缓存成为行业标准
2. **P/D 分离已不再是可选项**：99% 缓存时单 Prefill 实例服务 45--87 个 Decode 实例；DistServe/Splitwise 结论在 V4 上得到具体数字验证
3. **硬件应按阶段选型**：Prefill 计算受限选 910C，Decode 带宽受限选 H20；2026 年云服务商将提供分离 SKU
4. **网络带宽是 MoE Decode 的新瓶颈**：910C 7.8 倍 EP 带宽优势；256K Decode 时通信已占 22%，需将互联预算提升至 GPU 总成本的 30%
5. **算法-硬件协同设计不可或缺**：mHC 融合在 910C 上带来 7.95 倍加速，H20 上仅 1.94 倍；同一优化在不同硬件上回报差异高达 4 倍
6. **超长上下文变得触手可及**：1M token 时 V4 KV 仅 6.9 GB（V3 的 1/10）；单节点 TTFT 约 1 秒，对话级应用可行

### 幻灯片 21：附录 — 方法论与数据来源

- **Roofline 模型**：每个算子时间 = max(cube, vec, mem) + comm；瓶颈 = argmax(cube, vec, mem)
- **性能参数**：910C — Cube 40% 利用率，HBM BW 30% 利用率；H20 — Cube 50%，HBM BW 80%
- **FLOPs 约定**：矩阵乘法 [M,K]x[K,N] = M×N×K×2；BF16 = 2 字节，FP32 = 4 字节
- **搜索空间**：TP ∈ {1..64}，EP ∈ {8..256}，DP ∈ {1..8}，BS ∈ {1..512}；约束 (TP×DP) % EP == 0
- **通信模型**：AllReduce/AllGather/ReduceScatter 用 TP 带宽；AllToAll 用 EP 带宽；通信与计算顺序执行（保守上界）
- **Decode 快速模式**：周期采样 P=LCM(compress_ratios)=128，梯形插值；16 倍加速，误差<0.001%
- **P/D 分离（Schema v2）**：整数配平，balance_tolerance=0.1，hbm_reserved_pct=10%
- **数据来源**：`report/data/` 下 11 个 JSON 文件（search_results_910C/H20、hardware_comparison、pd_ratio_analysis、op_analysis、sp_comparison、mhc_optimization_comparison、kv_cache_scaling、attention_analysis、v3_comparison、long_context_1m_1k_prefix_cache）
